# Онлайн-мессенджер

Этот проект — простой веб-мессенджер на Node.js и SQLite с регистрацией, входом, списком чатов и отправкой сообщений.

## Файлы

- `index.html` — пользовательский интерфейс мессенджера
- `style.css` — стили для интерфейса
- `script.js` — клиентская логика (авторизация, загрузка чатов, отправка сообщений)
- `server.js` — сервер на Express + SQLite
- `database.sql` — схема базы данных
- `messenger.db` — рабочая база данных SQLite
- `package.json` — зависимости и команда запуска
- `FEATURES.md` — описание реализованных функций
- `SECURITY.md` — меры защиты и HTTP-заголовки безопасности

## Установка

1. Откройте терминал в папке проекта.
2. Установите зависимости:

```bash
npm install

npm install socket.io
```

## Запуск

```bash
npm start
```

После запуска сервер будет доступен по адресу:

```
http://localhost:3000
```

Если вы хотите открыть сайт с телефона или другого устройства в одной локальной сети, используйте IP-адрес компьютера, на котором запущен сервер:

```
http://<IP_вашего_компьютера>:3000
```

Вы также можете открыть сайт по простой ссылке:

```
http://localhost:3000/link.my
```

или с другого устройства:

```
http://<IP_вашего_компьютера>:3000/link.my
```

Чтобы узнать IP-адрес, выполните `ipconfig` (Windows) или `ifconfig` / `ip a` (Linux/macOS) и найдите адрес вашей Wi-Fi или Ethernet-сети.

> Устройства должны находиться в одной локальной сети.

Если сайт не открывается с другого устройства, проверьте брандмауэр и разрешите входящие соединения на порт `3000` для Node.js.

## Как пользоваться

1. Откройте в браузере `http://localhost:3000`.
2. Зарегистрируйтесь новым пользователем или войдите по email и паролю.
3. После входа появится список чатов и окно для отправки сообщений.
4. Нажмите `+`, чтобы создать новый чат.

## Дополнительные функции
- Загрузка аватара пользователя через `POST /api/user/avatar`
- Настройки профиля и уведомлений
- Поиск сообщений внутри чата и по списку чатов
- Отправка файлов и голосовых сообщений
- Ответы на сообщение и реакции на сообщения
- Приглашения в общий чат по коду: получение кода и присоединение по `POST /api/chats/join`
- Защита от копирования: мета-теги и серверные HTTP-заголовки безопасности
- Реальное время через WebSocket (Socket.IO): клиенты подписываются на чат по `joinChat`, новые сообщения рассылаются в комнату через `newMessage`

## Пользовательская информация

Информация о текущем вошедшем аккаунте выводится в интерфейсе после авторизации:
- имя пользователя
- уникальный код пользователя

Если нужно просмотреть все учетные записи напрямую, они хранятся в таблице `users` файла базы данных `messenger.db`.

## API

Основные маршруты сервера:

- `POST /api/register` — регистрация
- `POST /api/login` — вход
- `POST /api/logout` — выход
- `GET /api/auth` — проверить авторизацию
- `GET /api/user` — данные текущего пользователя
- `POST /api/user/avatar` — загрузка аватара пользователя
- `GET /api/chats` — список чатов
- `GET /api/messages/:chatId` — сообщения чата
- `POST /api/messages` — отправка текста в чат
- `POST /api/messages/file` — отправка файла/голосового сообщения
- `POST /api/chats` — создание нового чата с кодом приглашения
- `GET /api/chats/invite/:chatId` — получить код приглашения для чата
- `POST /api/chats/join` — присоединиться к чату по коду
- `DELETE /api/chats/:chatId` — удалить чат
- `PUT /api/messages/:messageId` — редактировать сообщение
- `DELETE /api/messages/:messageId` — удалить сообщение
- `POST /api/reactions` — добавить реакцию на сообщение
- `DELETE /api/reactions/:messageId/:emoji` — удалить реакцию
- `GET /api/search?q=...` — поиск по чатам и сообщениям

## Заметки

- Пароли хранятся в базе в хэшированном виде с помощью `bcryptjs`.
- Для корректной работы необходимо разрешение на запись в каталог проекта, чтобы сервер мог создать/обновить `messenger.db`.

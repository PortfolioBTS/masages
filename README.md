# Онлайн-мессенджер

Этот проект — простой веб-мессенджер на Node.js и SQLite с регистрацией, входом, списком чатов и отправкой сообщений.

## Файлы

- `index.html` — пользовательский интерфейс мессенджера
- `style.css` — стили для интерфейса
- `script.js` — клиентская логика (авторизация, загрузка чатов, отправка сообщений)
- `server.js` — сервер на Express + SQLite
- `database.sql` — схема базы данных
- `messenger.db` — рабочая база данных SQLite
- `package.json` — зависимости и команда запуска

## Установка

1. Откройте терминал в папке проекта.
2. Установите зависимости:

```bash
npm install
```

## Запуск

```bash
npm start
```

После запуска сервер будет доступен по адресу:

```
http://localhost:3000
```

## Как пользоваться

1. Откройте в браузере `http://localhost:3000`.
2. Зарегистрируйтесь новым пользователем или войдите по email и паролю.
3. После входа появится список чатов и окно для отправки сообщений.
4. Нажмите `+`, чтобы создать новый чат.

## Пользовательская информация

Информация о текущем вошедшем аккаунте выводится в интерфейсе после авторизации:
- имя пользователя
- уникальный код пользователя

Если нужно просмотреть все учетные записи напрямую, они хранятся в таблице `users` файла базы данных `messenger.db`.

## API

Основные маршруты сервера:

- `POST /api/register` — регистрация
- `POST /api/login` — вход
- `POST /api/logout` — выход
- `GET /api/auth` — проверить авторизацию
- `GET /api/user` — данные текущего пользователя
- `GET /api/chats` — список чатов
- `GET /api/messages/:chatId` — сообщения чата
- `POST /api/messages` — отправка сообщения
- `POST /api/chats` — создание нового чата

## Заметки

- Пароли хранятся в базе в хэшированном виде с помощью `bcryptjs`.
- Для корректной работы необходимо разрешение на запись в каталог проекта, чтобы сервер мог создать/обновить `messenger.db`.
